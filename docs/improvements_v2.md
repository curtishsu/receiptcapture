Overall improvements


1. Overview
The tabs at the bottom should persist on all screens. Currently, if you go to a receipt, the tabs at the bottom aren't present
2. For the 'amount' when editing it is currently in increments of .01. The increments should be .25. 


# Receipt Capture
1. On a given receipt, a user might 'x' out line items because they aren't relevant to groceries or because it was purchased for another user. When this 'x' out happens, deduct the value from the total amount
2. Sometimes items purchased may include tax. Because tax isn't an item, it can be captured in the total next to the actual total. At the top where there is a total: example (2026-04-02 • $32.39) have the tax in parantheses. For example: 2026-04-02 • $32.39 (3.10). 
3. For quantity, there are two ways that an item can have multiple quantities. 1. The same item appears multiple time. 2. Underneath the item there is a small section that says 'quantity 2' 
4. Instead of having the word 'Blank' for items that can be identified from the receipt, have it just be blank. When saved, have them be dashes. 


# Receipt History
## View Receipt
1. Receipt ID and Items are currently in a box when viewing a past receipt. These do not need to be there. Instead, have The items be next to the price and separated by a dot (e.g. 10 items). The receipt id doesn't need to be shown except when in edit mode. In edit mode, it should be under 'Edit Receipt' Header
2. When clicking 'back' on a receipt, it should take me back to the Receipt history page, not the overall landing page
3. The drop down for the unit selector isn't intuitive. The design looks like it's an autofill suggestion, not coming from the actual drop down menu


## Edit Receipts
1. When editing a receipt, I expect unit to be a dropdown that contains all the past units. The dropdown menu should be broken into two headers. 1. Unit (containing lb, oz etc.) and then 'per unit'. This is the "/pound", etc. The unit one should be above. 
2. When clicking save', the 'Receipt updated.' confirmation is at the top. It should be at the bottom about the 'Save Changes' button
